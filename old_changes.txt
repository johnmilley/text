Shipped 2026-07-12 (mobile keyboard + highlight + housekeeping):
- ==highlight== now renders everywhere (editor, preview, publish/PDF, web
  build), i=. Every theme gets a `highlight` color token — your
  keepers each got a tuned wash (github-light uses GitHub's actual marker
  color). `==**bold** inside==` works; `a == b` stays literal; never in code.
- md keyboard bar remodeled on your iA screenshot: # (each tap deepens the
  heading — the H-toggle is gone), B I ` A(highlight, wears the theme wash)
  [[ ]] • ☑ ↺ ↻, and a ⌘ sheet holding the second-string stuff: search, link,
  quote, strikethrough (backtick took its old slot in the main row), hide
  keyboard. Positioning now tracks the keyboard every frame — the event-
  driven version was what let it strand mid-screen on iOS.
- desktop image paste: when WebKitGTK hands the editor an empty paste event
  (how clipboard screenshots often arrive on Linux), the app now reads the
  image straight from the native clipboard (new read_clipboard_image command,
  arboard). Same pasted-<timestamp>.png naming as mobile.
- fullscreen slides in light mode: the deck's background rule was riding a
  selector list with :-webkit-full-screen, which Firefox rejects wholesale —
  so the deck went transparent over the black fullscreen backdrop. Split the
  rules; slides now keep their paper color (and it was invisible in dark mode
  because black-on-black).
- publish button left the file bar; still on every folder right-click,
  Ctrl+Shift+S, and now settings → help.
- themes trimmed 39 → 19: text pair, ia-writer pair, emotion pair,
  monokai-calm, dracula, nord/nord-light, gruvbox/gruvbox-light, solarized
  pair, catppuccin mocha/latte, tokyo night/day, github-light. Retired ones
  vanish from your themes folder ONLY if you never edited them (an edited
  copy is yours and stays; it keeps working as a custom theme).
- dataview grew WHERE: `TABLE status, due FROM #course WHERE status = active
  AND due <= 2026-09-01 SORT due` — ops = != > < >= <= contains, AND-chained.
- file manager: added "copy to folder…" (native picker) — that's also the
  answer to passing files between instances: copy into the other window's
  folder and its watcher shows it instantly. (Tree drag-to-move between
  folders already existed, pointer-based. A real cross-window drag can't be
  done reliably in webkit2gtk — HTML5 dnd fights the native file-drop hook.)
- also shipped, from the older text_changes.md list: corkboard (right-click a
  folder or Ctrl+Shift+D — index cards w/ frontmatter synopses, drag to
  reorder, drill into subfolders) and window position/size now persist across
  launches (Wayland note: compositors get final say on position; size always
  restores).

Shipped 2026-07-07 (folding + brighter chrome + icon crop; 0.2.9):
- foldable markdown headings: chevrons in a slim gutter left of the text fold
  a heading's whole section (##'s section ends at the next ##/#); code fences,
  lists, quotes, and multi-line paragraphs fold too. Folded text collapses to
  a clickable ⋯ pill. Ctrl+Shift+[ / Ctrl+Shift+] fold/unfold at the cursor
  (heading line), Ctrl+Alt+[ / ] fold/unfold everything. Folds survive tab
  switches; edits above a fold leave it alone.
- brighter chrome: toolbar/tab/title-bar/tree-folder text was fg-muted, which
  most themes tune for de-emphasis, not for UI you read constantly. A derived
  --fg-chrome token (fg-muted nudged 42% toward fg, so it brightens dark
  themes and darkens light ones) now colors the title bar, sidebar tabs and
  footer, folder row, tree dirs, tab bar, pane/preview/image/pdf toolbars,
  and calendar buttons. Hints and captions stay muted. Themes can override
  --fg-chrome in their raw css.
- app icon zoomed in: the tiled-cats source is cropped to its center 70%, so
  the cats actually read at taskbar sizes. All sizes regenerated via
  `tauri icon` from the cropped _source-icon.png.

Shipped 2026-07-03 (launch speed + line-width rethink; 0.2.8):
- launch no longer waits for the whole vault: the last session's tabs open
  immediately while the folder scan, file watcher, and config save run in the
  background. Tabs whose files vanished while the app was closed are pruned
  once the scan lands (the active one falls back to the first note).
- the vault scan, search, and backlinks moved off the app's main thread — on
  Tauri, synchronous commands run on the UI thread, so a big folder froze the
  window at launch and on every autosave-triggered rescan. Unknown-extension
  text sniffing (an 8 KiB read per file, per rescan) is now cached by mtime.
- side effect worth knowing: a tab whose file still exists but sits outside
  the (possibly vanished) folder now reopens as a visitor tab instead of
  being dropped.
- margins/wrapping rethink: text now wraps at a line width (default 80
  characters) and the column centers when the window is wider — no more
  edge-to-edge lines on a big monitor. settings → appearance → "line width"
  (config line_width; 0 = fill the window, the old behaviour).
  editor_margin remains as the minimum edge spacing. The same measure
  applies to the split pane and the markdown preview; zen mode caps at
  68ch as before (narrower if your line width is narrower).

Shipped 2026-06-30 (open loose files + new-tab page + zen typewriter + KDE icon):

  Alt+O opens any file (even outside the current folder) in a new tab — the
  plain-notepad half of the app. A file from outside the folder is a "visitor"
  tab: italic label with a ↗ marker, never joins the tree or backlinks. With no

- customizable new-tab page: a _newtab.md at the folder root renders as the

- new "iA Writer Dark" theme (near-black paper, warm grey ink, calm blue accent).
- zen mode settings (settings → zen mode): "keep sidebar in zen" shows the file
  tree in fullscreen; "typewriter scrolling" on/off; "cursor position" upper
  third (default) or middle. Config keys zen_sidebar, zen_typewriter,
  typewriter_anchor.
- typewriter no longer fights text selection: it only recenters on a plain
  caret, so dragging or shift-selecting a range holds still (recentring resumes

- Linux/KDE app icon fix: the icon name "text" collided with Breeze's built-in

Shipped 2026-06-24 (wysiwyg line breaks):
- new setting "single line breaks" (settings → editing; config key
  single_line_breaks, off by default). When on, every newline in the source
  renders as a <br> in the preview/exported HTML instead of CommonMark's
  default of joining wrapped lines into one paragraph — so the rendered output
  matches the editor line-for-line. Applies to preview, published sites, and
  PDF (all go through the one pulldown-cmark renderer).
